# V7.3 Runtime 3A Runbook

V7.3 runtime_events are offline research cache files for historical runtime A2/A3.
They are not live-trading dependencies.

Normal research entry:

```bash
python tools/backtest_local_data.py \
  --symbol ETH-USDT-SWAP \
  --trades-dir data/okx/raw/trades/ETH-USDT-SWAP \
  --books-dir data/okx/raw/books/ETH-USDT-SWAP \
  --run-name V730_1d_20260401 \
  --generate-reports \
  --kline data/history_k/ETH-USDT-SWAP_1m_20260401_0000_20260525_2359.csv \
  --start-date 2026-04-01 \
  --end-date 2026-04-01
```

`backtest_local_data.py` is the normal research entry point. Use
`--start-date YYYY-MM-DD` and `--end-date YYYY-MM-DD` for the normal time range.
It automatically selects matching trades/books files, checks
`data/derived/runtime_events`, reuses existing day shards, generates missing day
shards while the trade stream is already being replayed, writes only
`runtime_events_ref.json` under `reports/backtests/<run>/`, and passes the cache
directory to zone_truth runtime 3A report generation.

ETH-USDT-SWAP uses the OKX known default `contract_multiplier = 0.1`, so normal
ETH research commands do not need `--contract-multiplier`. If you research
another SWAP symbol, add it to the known defaults first or pass
`--contract-multiplier` explicitly; the system must not guess unknown SWAP
contract sizes. Non-SWAP symbols default to `1.0`.

Cache location:

```text
data/derived/runtime_events/<symbol>/<param_key>/
  YYYY-MM-DD.jsonl
  runtime_events_manifest.json
  runtime_events_cache_meta.json
```

`param_key` includes bucket, rolling window, contract multiplier, notional mode,
and schema version, for example
`bucket_1s_rolling_3s_cm_0p1_nm_price_x_size_x_contract_multiplier_sv_v7p3_runtime_events_1`.
Manifest and cache meta record the same parameters, so incompatible shards are
not reused.

Reuse and generation:

- Existing valid day shards are reused.
- Missing or invalid day shards are generated during replay.
- `reports/backtests/<run>/runtime_events_ref.json` records selected days,
  cache hits, misses, generated days, cache parameters, contract multiplier
  source, and the cache directory.
- Runtime events are not copied into report directories.

MVP_VP_3A_LITE:

- This is a paper/live candidate strategy kernel, not a complete live order
  system.
- It is emitted inside the existing zone_truth runtime 3A report family.
- Output files are `zone_truth_3a_mvp_trades.csv`,
  `zone_truth_3a_mvp_summary.json`, `zone_truth_3a_mvp_decision.md`,
  `zone_truth_3a_mvp_by_vp_setup.csv`, and
  `zone_truth_3a_mvp_by_a2_quality.csv`.
- Default MVP trading setup is only `SELL_ABOVE_VAH_ABSORB`.
- `BUY_BELOW_VAL_ABSORB` and `BUY_NO_VP_EDGE` are observe-only and do not enter
  MVP trades.
- `SELL_INSIDE_VALUE_ABOVE_POC_ABSORB` and `SELL_NO_VP_EDGE` are blocked from
  MVP trades.
- MVP readiness uses only default 900s expiry, deduplicated one signal one
  trade. It does not use best expiry or all-expiry variants.
- `live_readiness_gate` reports `NOT_READY`, `PAPER_READY`, or
  `SMALL_SIZE_READY`. `NOT_READY` is expected for small or weak samples.

Recommended MVP validation sequence:

1. Run 1 day and check `cache_miss_days` then `generated_days`.
2. Rerun the same 1 day and confirm `cache_hit_days`.
3. Run 3 days and verify MVP output files and `zone_truth_3a_mvp_decision.md`.
4. Run 7 days and look for positive MVP expectancy hints.
5. Run 30 days only if 7 days remains promising.
6. Enter paper trading only after 30 days passes `PAPER_READY`.
7. Consider small-size live only after paper results pass review.

Do not jump directly to 30 or 50 days.

Advanced debug parameters:

To force rebuild for selected days:

```bash
python tools/backtest_local_data.py ... \
  --runtime-events-cache-overwrite true
```

To override cache days explicitly:

```bash
python tools/backtest_local_data.py ... \
  --runtime-events-cache-days 2026-04-01,2026-04-02
```

`--runtime-events-cache-days` is only an advanced/debug override for
runtime_events cache days. Do not use it in normal runs; use `--start-date` and
`--end-date` instead.

To clean cache, remove only the parameter directory you intend to rebuild:

```bash
rm -rf data/derived/runtime_events/ETH-USDT-SWAP/bucket_1s_rolling_3s_cm_0p1_nm_price_x_size_x_contract_multiplier_sv_v7p3_runtime_events_1
```

The standalone tool remains available for debug or manual rebuilds:

```bash
python tools/build_runtime_events_from_okx_trades.py \
  --symbol ETH-USDT-SWAP \
  --trades-dir data/okx/raw/trades/ETH-USDT-SWAP \
  --overwrite
```

Do not run it for normal research. Use it only when rebuilding cache without a
books replay, or when debugging raw trade parsing.

Metrics to watch:

- `peak_rss_mb`
- `runtime_window_reads`
- `runtime_max_window_ticks`
- `runtime_performance_warning`
- `cache_hit_days`
- `generated_days`

Live mode:

```text
OKX WebSocket trades
-> RuntimeEventAccumulator.update_trade()
-> active_buy_notional_3s / active_sell_notional_3s / cvd_delta_3s / price_velocity
-> runtime A2/A3
```

Live trading uses `RuntimeEventAccumulator` in memory and does not read
runtime_events files. Offline runtime_events are research cache files: the
historical persisted form of the same rolling orderflow accumulator.
