# V7.3 Runtime 3A Strategy

V7.3.0 separates future quality labels from runtime entry logic.

Runtime A2 is an event-driven state machine after A1: defended level holds, active pressure exhausts, price stays in a small quiet box, and CVD stops worsening. It can become `A2_READY_FOR_A3` after a few seconds if `A2_RT_MIN_QUIET_SEC` and `A2_RT_MIN_TICK_COUNT` are satisfied. Expiry is swept across 180, 300, 600, 900, 1200, and 1800 seconds.

Runtime A3 is immediate box breakout: price breaks the A2 quiet box, same-side active notional bursts, active flow dominates the opposite side, CVD aligns, and price velocity confirms. It does not wait for 3m/5m persistence.

VP is A1 absorption context, not an A3 current-price hard filter. Direction-aware fields identify VAL/VAH/POC, HVN/LVN above and below, A1 value location, node location, and setup classes such as `BUY_NEAR_VAL_ABSORB`, `BUY_LVN_BELOW_HVN_ABSORB`, `SELL_NEAR_VAH_ABSORB`, and `SELL_LVN_ABOVE_HVN_ABSORB`.

Targets record fixed 2R plus POC, directional HVN, opposite value edge, next LVN, and a hybrid minimum-2R candidate. The first runtime report keeps fixed 2R as the main target model and records the VP candidates for analysis.

Future/offline/sim fields remain useful for outcome analysis only. They are blocked from entry by the no-future audit.
