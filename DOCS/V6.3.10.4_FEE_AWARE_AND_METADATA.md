# V6.3.10.4 Fee-Aware A1 Edge Metrics and Metadata

## Scope

V6.3.10.4 changes only the offline A1 Edge Validation Suite. It does not change A1 runtime event generation, `a1_reaction_events.jsonl` epoch semantics, real trading, virtual positions, A2, or A3.

## Fee Model

Default:

```text
roundtrip_fee_pct = 0.001
```

`0.001` means 0.1% roundtrip notional fee.

For each event:

```text
fee_u = entry_price * roundtrip_fee_pct
fee_share_r = fee_u / risk_u
net_directional_mfe_r = directional_mfe_r - fee_share_r
net_directional_mae_r = directional_mae_r + fee_share_r
```

Raw +1R does not equal net +1R. Raw +1R only means price moved one risk unit before fees. Net +1R requires price movement to exceed `1R + fee_share_r`.

裸 +1R 只是价格走了一个风险单位，不代表扣费后赚到 +1R。净 +1R 必须覆盖 `1R + 手续费折算成的 R`。

If `risk_pct` is smaller than `roundtrip_fee_pct`, the fee can be larger than 1R. This is expected and is not clipped.

Example:

```text
entry_price = 2130
roundtrip_fee_pct = 0.001
fee_u = 2.13
```

If:

```text
risk_u = 1.0
fee_share_r = 2.13R
```

Then after price moves raw +1R:

```text
net_mfe_r = 1.0 - 2.13 = -1.13R
raw hit_plus_1r = True
net_hit_plus_1r = False
```

## Edge Decision Basis

All A1 edge decisions now use fee-aware net R fields:

- `a1_avg_net_mfe_r_15m`
- `random_avg_net_mfe_r_15m`
- `a1_net_hit_1r_rate_15m`
- `random_net_hit_1r_rate_15m`
- `net_mfe_edge_15m`
- `net_hit_1r_edge_15m`

Raw fields are still exported for diagnostics:

- `directional_mfe_r`
- `hit_plus_1r`
- `a1_avg_raw_mfe_r_15m`
- `random_avg_raw_mfe_r_15m`
- `raw_mfe_edge_15m`

Raw fields must not be used for Go / No-Go. A1 vs random Go / No-Go is based on net MFE_R and net hit rate. Volatility/range can be reported, but range alone no longer promotes A1 to GO.

## Reproducibility Metadata

Each run writes:

```text
reports/a1_edge/a1_run_metadata.json
```

The same metadata is merged into:

```text
reports/a1_edge/a1_edge_summary.json
```

The markdown report shows the key metadata at the top:

- events file path/name/size/hash/non-empty line count
- K-line file path/name/size/hash/CSV data row count
- analyzer version
- git commit
- Python version
- output directory
- created UTC time
- symbol, windows, seed, random sample count
- risk settings, baseline risk mode, K-line timezone
- `roundtrip_fee_pct`

This prevents a report from becoming detached from the JSONL and K-line CSV that produced it. It also prevents comparing a report to the wrong K-line file after re-exporting or timezone correction.

## CLI

```bash
python tools/analyze_a1_edge.py \
  --events logs/research/a1_reaction_events.jsonl \
  --klines data/history_k/ETH-USDT-SWAP_1m_20260521_2000_20260522_2000.csv \
  --out reports/a1_edge \
  --symbol ETH-USDT-SWAP \
  --random-samples-per-event 10 \
  --seed 42 \
  --kline-timezone Asia/Shanghai \
  --roundtrip-fee-pct 0.001
```

The CLI prints:

```text
[A1-EDGE-FEE] roundtrip_fee_pct=0.001 fee_model=roundtrip_notional_pct
```

The suite remains offline-only.
