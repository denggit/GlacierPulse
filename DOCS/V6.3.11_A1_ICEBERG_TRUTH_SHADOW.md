# V6.3.11 A1 Iceberg Truth Shadow System

V6.3.11 is a research and shadow-audit version. It is not a trading version, not an A2/A3 implementation, and not a dynamic-parameter takeover release.

The current A1 runtime decision path remains unchanged:

```text
tick trade -> pending event -> book settlement -> IcebergDetector result -> ZoneTracker -> A1 Reaction
```

The new system only adds a side-channel observer after Phase1 settlement. Every settled Phase1 candidate is recorded, including `ICEBERG`, `IGNORE`, `SPOOFING`, and `CANCEL`. The existing A1 `ICEBERG / IGNORE / SPOOFING / CANCEL` judgment is not changed by this release.

## What The Shadow System Does

Phase1 Candidate Recorder writes `logs/research/phase1_candidates.jsonl`.

It writes two record types:

- `candidate_settled`: emitted immediately after a pending event settles.
- `candidate_finalized`: emitted after post-settlement observation and truth scoring.

Iceberg Truth Score evaluates whether a candidate behaved like a real iceberg after settlement. The score is research-only and never feeds back into A1 runtime decisions.

True iceberg behavior is not primarily judged from 1m K-lines. A real A1 iceberg needs tick and book evidence in short post windows: attack strength, impact suppression, hidden absorption, reload/replenish, sweep reclaim or failed breakout, CVD-price divergence, and non-acceptance outside the zone.

## Post Windows

The tracker observes tick and book summaries after settlement for:

- 1s
- 5s
- 30s
- 120s

It stores only summaries, not full book snapshots. If the tracker fails, it logs `[PHASE1-TRUTH-FALLBACK]` and the A1 main flow continues.

## Parameter Grid Calibration

`tools/analyze_phase1_candidates.py` reads finalized candidate records and produces:

- `phase1_truth_summary.md`
- `phase1_candidate_events.csv`
- `phase1_truth_by_result.csv`
- `phase1_truth_by_session.csv`
- `phase1_truth_by_quality.csv`
- `phase1_truth_by_behavior.csv`
- `phase1_parameter_grid.csv`
- `phase1_parameter_grid_by_session.csv`
- `phase1_dynamic_preview_summary.json`

The grid is for later parameter selection. It does not alter live A1 thresholds.

## Dynamic Parameter Preview

Dynamic preview writes `runtime_state/a1_dynamic_params.json` with `mode=preview_only`.

`active_params.source` is always `static`, and `safety.dynamic_params_active` is always `false` in this version. The preview JSON is observation-only and does not participate in A1 judgment.

Static-vs-dynamic shadow comparison is reserved for a later version.

## Safety

Real trading remains closed. Virtual trading is not enabled by V6.3.11. The release does not change:

- `a1_reaction_events.jsonl` semantics
- A1 Edge Analyzer semantics
- OKX tick/book multipliers
- historical K-line loader
- A1 runtime ICEBERG/IGNORE/SPOOFING/CANCEL decisions
