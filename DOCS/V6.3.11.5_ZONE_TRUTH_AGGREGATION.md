# V6.3.11.5 Zone Truth Aggregation

V6.3.11.5.1 adds cleanup fields for full pie membership, final reaction tracking, complete-only forward averages, and runtime-aligned session tags.

V6.3.11.5.2 upgrades the zone truth schema version and exposes forward calculation anchors and entry-price sources in the report.

## Scope

This version adds offline zone-level research for Phase1 truth data and A1 reaction zones. It does not change A1 runtime decision rules, trading flags, OKX multipliers, or historical K-line volume semantics.

## Concepts

- `pie` is one Phase1 absorption event, recorded as a Phase1 candidate with an `event_key`.
- `zone` is an A1-generated iceberg price area, identified by `zone_id`.
- A1 discovers pies and creates or updates zones.
- A2 observes zones. The A2 object is a zone, not a single pie.
- `A2_PRE_POOL` means observation pool only. A zone is eligible when `iceberg_pie_count >= 1`.
- Truth Score is a post-event audit label for pies. It is not a realtime A2_PRE_POOL condition.
- MFE/MAE are post-event research results. They are not realtime A2_PRE_POOL conditions.

## What This Version Implements

- Aggregates pie-level Phase1 truth into zone-level truth profiles.
- Lets runtime candidate records carry additive `zone_id` / `assigned_zone_*` fields when the current flow can directly identify the updated zone.
- Lets A1 zones retain additive `pie_event_keys` for direct pie membership visibility.
- Produces `zone_truth_events.csv`, one row per zone.
- Emits full pipe-delimited pie membership fields such as `pie_event_keys` and `iceberg_pie_event_keys`.
- Computes 15m / 1h / 4h zone-level forward MFE and MAE from 1m K-lines.
- Emits `forward_anchor_ts`, `forward_anchor_source`, `forward_anchor_local_time`, `forward_entry_price`, and `forward_entry_price_source` so later analysis can audit exactly where forward metrics started.
- Produces grouped reports by primary reaction, final reaction, direction, session, truth bucket, match quality, and A2_PRE_POOL eligibility.

## What This Version Does Not Implement

- It does not implement `A2_VALIDATED`.
- It does not implement A3.
- It does not implement A2 compression, retest, sweep reclaim, or active-trade-decay validation.
- It does not change A1 ICEBERG / IGNORE / SPOOFING / CANCEL runtime judgment.
- It does not let Truth Score, MFE/MAE, or dynamic preview parameters affect A1 or A2_PRE_POOL.
- It does not enable real trading or virtual trading.

## Matching

Exact match has priority. If a candidate has `zone_id`, `assigned_zone_id`, `frozen_zone_id`, or `matched_zone_id`, and that id exists in A1 reaction data, the pie is assigned by exact match.

Fuzzy match is only a compatibility path for old data. It requires direction agreement plus time and price proximity. It scores price-in-zone, zone overlap, event-time proximity, and ICEBERG result. A score below 50 is not accepted.

Unmatched ICEBERG pies are not dropped. They become synthetic zones with `zone_id = synthetic-{event_key}` and `zone_source = synthetic_from_pie`.

Synthetic zones are currently emitted per unmatched ICEBERG pie. They do not represent a true merged synthetic price zone. A future version may add `synthetic_merge_enabled` to combine nearby unmatched ICEBERG pies.

Future runtime should prefer direct pie-to-`zone_id` recording so fuzzy matching becomes unnecessary.

## Forward Metrics Anchor

Zone forward metrics default to the existing anchor order:

1. `reaction_event_ts`
2. `frozen_ts`
3. `best_pie_ts`
4. `first_seen_ts`

The report exposes the resolved value in `forward_anchor_ts` and the selected field name in `forward_anchor_source`. The local timestamp is also emitted as `forward_anchor_local_time`.

Entry price uses the existing order:

1. `zone_mid`
2. `best_pie_price`
3. `settle_price`

The report exposes the resolved price in `forward_entry_price` and the selected field name in `forward_entry_price_source`.

For zones with multiple reactions, `final_reaction_type` is only the final reaction label. It does not mean forward metrics start from `final_reaction_ts`. A later version can add `final_reaction_forward_metrics` if that research view is needed.

## A2_PRE_POOL Research Rule

`a2_pre_pool_eligible = True` only when a zone has at least one `result=ICEBERG` pie.

The rule intentionally ignores:

- Truth Score
- MFE/MAE
- CLEAN_HOLD / FAILED_RECLAIM reaction type
- future K-line data

These fields are for offline research into which pre-pool zones may deserve later `A2_VALIDATED` rules.
