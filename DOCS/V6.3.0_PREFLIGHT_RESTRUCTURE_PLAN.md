# V6.3.0-preflight: Semantic Restructure Plan

## 1. Current Reality

Current V6.2.9.5 is not a complete Fabio TripleA implementation. The live runtime is closer to:

- A1 iceberg absorption research
- A1 frozen zone lifecycle
- A1 reaction evaluator
- execution research candidate
- virtual position simulation
- outcome research

The names Phase1, Phase2, and Phase3 remain in the runtime code for compatibility, but their current meanings do not map cleanly to Fabio-style A1/A2/A3.


## 7. V6.3.1 Schema Adapter Direction

- V6.3.1 adds read-only A1 schema dataclasses and adapters.
- These adapters normalize old runtime dicts into A1 semantic names.
- Existing runtime dicts and legacy fields remain unchanged.
- The adapters do not participate in live trading decisions yet.
- A1OutcomeRecord normalizes IcebergOutcomeEvaluator fields (`mfe` / `mae`) and Phase3 virtual outcome fields (`max_favorable_u` / `max_adverse_u`) into `mfe_u` / `mae_u`.

## 8. V6.3.2 A1 Reaction Implementation Migration

- V6.3.2 moves the A1 reaction evaluator implementation to `src/strategy/a1_reaction/reaction_evaluator.py`.
- The legacy path `src/strategy/phase2_orderflow_evaluator.py` remains as a deprecated compatibility wrapper.
- Runtime behavior, PHASE2 log tags, state names, and `phase2_type` fields remain unchanged.
- This is the first step toward making the V6.3 semantic architecture the primary implementation path.
- No A2 or A3 logic is implemented in this version.

## 9. V6.3.3 Execution Research Candidate Implementation Migration

- V6.3.3 moves the execution research candidate/risk-gate implementation to `src/strategy/execution_research/candidate_evaluator.py`.
- The legacy path `src/strategy/phase3_candidate_evaluator.py` remains as a deprecated compatibility wrapper.
- Runtime behavior, PHASE3 log tags, candidate decisions, risk sizing, `phase2_type`, and `candidate_type` fields remain unchanged.
- This migration prevents the current execution research layer from being confused with future Fabio-style A3 aggression/ignition.
- No A2 or A3 logic is implemented in this version.
