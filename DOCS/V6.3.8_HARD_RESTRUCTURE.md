# V6.3.8 Hard Semantic Restructure

V6.3.8 is a hard semantic restructure. It is not a strategy logic upgrade and
does not implement the full Fabio 3A pipeline.

The long-term strategy layout is now:

- `src/strategy/a1_absorption/`
- `src/strategy/a2_accumulation/`
- `src/strategy/a3_aggression/`
- `src/strategy/execution_research/`

The old phase1/phase2/phase3 wrapper paths have been removed. `A1AbsorptionEngine`
is the real implementation and is no longer a `Phase1Engine` alias.

`A1ReactionEvaluator` is A1 internal reaction research. It is not true Fabio A2.
`CandidateRiskEvaluator` is an execution research risk gate. It is not true Fabio
A3.

The A1 score model has been removed. `A1ResearchReportBuilder` now consumes raw
A1 mappings and aggregates original A1 absorption, reaction, candidate, and
outcome fields instead of `A1ScoreRecord`.

A2/A3 directories are skeletons only in this version. They do not participate in
runtime candidate generation, virtual positions, or trading.

A1 reaction confirmed events enter execution research candidate logging by
default, but they do not enter virtual positions unless
`A1_REACTION_TO_VIRTUAL_POSITION_ENABLED=True`.

Real trading remains disabled by default:

- `REAL_EXECUTION_ENABLED=False`
- `PHASE3_REAL_TRADING_ENABLED=False`
- `A1_SINGLE_EVENT_TRADING_ENABLED=False`

This version is still not the complete Fabio 3A implementation. V6.4 is the
planned starting point for true A2 Accumulation / Compression / Liquidity
Vacuum. V6.5 is the planned starting point for true A3 Aggression / Ignition.

V6.3.8.1 polishes this restructure by tightening A1 semantic internal names,
component status logs, A1 candidate log fields, schema adapters, tests, and
documentation without changing strategy behavior.
