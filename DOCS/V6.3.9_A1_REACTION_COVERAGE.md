# V6.3.9 / V6.3.9.1 A1 Reaction Coverage

## V6.3.9.1 fixes
- Added `_append_research_event` runtime helper and dedup key for `zone_id+a1_reaction_type+event_kind`.
- Added `ResearchRuntimeMonitor._a1_reaction_research_summary()` with exception-safe fallback.
- Added `CandidateRiskEvaluator._update_counters()` and verified summary counters.
- Added/strengthened tests for runtime path, monitor counters, taxonomy, recorder, engine drain.
- Migrated A1 reaction coverage configs to env-helper style.

## Event paths
- `confirmed_events`: consumed by candidate evaluator path (`PHASE2-CONFIRMED` chain).
- `research_events`: recorder/logging only; includes confirmed/failed/timeout/no_response/sweep_no_reclaim/reclaim_no_retest/missed_fast_move.
- `research_events` **do not enter** candidate evaluator or virtual position manager.

## Fast move
- Fast move is implemented as **research-only** (`MISSED_FAST_MOVE`), no trading side effects.

## Safety defaults
- `A1_REACTION_TO_VIRTUAL_POSITION_ENABLED=False` by default.
- No A2/A3 implementation in this patch.
