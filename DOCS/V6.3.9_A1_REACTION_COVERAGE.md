# V6.3.9 / V6.3.9.1 / V6.3.9.2 A1 Reaction Coverage

## V6.3.9.2 polish
- `A1_REACTION_RESEARCH_COVERAGE_ENABLED` is the total switch for A1 reaction research coverage. When disabled, research event generation, recorder drain, and `[A1-REACTION-RESEARCH-EVENT]` logs are skipped without touching the confirmed event path.
- Fast move research coverage now supports `A1_REACTION_FAST_MOVE_MIN_ACTIVE_NOTIONAL_3S`; the default `0.0` keeps previous behavior.
- `[V62-CONFIG-SNAPSHOT]` includes all V6.3.9 A1 reaction coverage, fast move, recorder, and research-event log controls.
- `src.strategy.a1_absorption` exports `A1ReactionEventRecorder`, taxonomy constants, and taxonomy helper functions.
- `research_events` remain recorder/logging only and still do not enter `CandidateRiskEvaluator` or `ResearchVirtualPositionManager`.
- `A1_REACTION_TO_VIRTUAL_POSITION_ENABLED` remains `False` by default.

## V6.3.9.1 fixes
- Added `_append_research_event` runtime helper and dedup key for `zone_id+a1_reaction_type+event_kind`.
- Added `ResearchRuntimeMonitor._a1_reaction_research_summary()` with exception-safe fallback.
- Added `CandidateRiskEvaluator._update_counters()` and verified summary counters.
- Added/strengthened tests for runtime path, monitor counters, taxonomy, recorder, engine drain.
- Migrated A1 reaction coverage configs to env-helper style.

## Event paths
- `confirmed_events`: consumed by candidate evaluator path (`PHASE2-CONFIRMED` chain).
- `research_events`: recorder/logging only; includes confirmed/failed/timeout/no_response/sweep_no_reclaim/reclaim_no_retest/missed_fast_move.
- `research_events` **do not enter** candidate evaluator or virtual position manager.

## Fast move
- Fast move is implemented as **research-only** (`MISSED_FAST_MOVE`), no trading side effects.

## Safety defaults
- `A1_REACTION_TO_VIRTUAL_POSITION_ENABLED=False` by default.
- No A2/A3 implementation in this patch.
