# V6.2 Research Shadow Run Guide

## Purpose

V6.2 is a Research Evaluator. It is not Real Trading.

The goal is to observe the full research loop from A1 frozen zones to Phase2 confirmed events, Phase3 candidates, Virtual Position open/close events, and Outcome summaries.

## Recommended Modes

Research mode:

```text
PHASE3_REAL_TRADING_ENABLED=False
REAL_EXECUTION_ENABLED=False
VIRTUAL_POSITION_MANAGER_ENABLED=True
VIRTUAL_SHADOW_MODE=False
```

Future shadow mode:

```text
PHASE3_REAL_TRADING_ENABLED=False
REAL_EXECUTION_ENABLED=False
VIRTUAL_POSITION_MANAGER_ENABLED=True
VIRTUAL_SHADOW_MODE=True
```

Real trading mode is out of scope for V6.2:

```text
REAL_EXECUTION_ENABLED=True
VIRTUAL_POSITION_MANAGER_ENABLED=False
```

Do not enable real trading in V6.2.

## Parallel Run Recommendation

- Keep the stable V6.1.3 process running.
- Start a separate V6.2 research process.
- Use different log files.
- Run both for at least 24 hours.
- 48-72 hours is recommended.
- Compare A1 frozen, Phase2 confirmed, Phase3 candidate, Virtual open/close, and Outcome summary logs.

## Startup Safety Logs

```bash
grep -E "V62-SAFETY|V62-COMPONENT|V62-CONFIG" nohup.out
```

Acceptance:

- Has `[V62-SAFETY-CHECK-PASSED]`.
- No `[V62-SAFETY-CHECK-FAILED]`.

## Phase2 Logs

```bash
grep "PHASE2-REGISTERED" nohup.out
grep "PHASE2-STATE" nohup.out
grep "PHASE2-CONFIRMED" nohup.out
```

## Phase3 Logs

```bash
grep "PHASE3-CANDIDATE" nohup.out
```

## Virtual Position Logs

```bash
grep "VIRTUAL-POSITION-OPEN" nohup.out
grep "VIRTUAL-POSITION-CLOSE" nohup.out
grep "VIRTUAL-STOP-UPDATE" nohup.out
grep "VIRTUAL-SUPPORT-UPDATE" nohup.out
```

## Outcome Logs

```bash
grep "PHASE3-OUTCOME" nohup.out
grep "PHASE3-OUTCOME-SUMMARY" nohup.out
```

## Runtime Heartbeat

```bash
grep "V62-HEARTBEAT" nohup.out
grep "V62-FINAL-SUMMARY" nohup.out
```

Key fields:

- `virtual_total_opened`
- `virtual_total_closed`
- `outcome_total_closed`
- `outcome_win_rate`
- `outcome_avg_realized_r`
- `outcome_profit_factor_r`
- `outcome_avg_mae_abs_r`
- `real_execution_enabled`
- `phase3_real_trading_enabled`

## Error / Forbidden Logs

```bash
grep -E "ERROR|FAILED|Traceback" nohup.out
grep -E "ORDER|OCO|REAL-TRADING|process_signal|_request" nohup.out
```

`Traceback` should not appear. Real order-related logs should not appear. If `[V62-SAFETY-CHECK-FAILED]` appears, stop this run and inspect the configuration before continuing.

## Acceptance Criteria

- No Traceback.
- No real trading logs.
- Has `V62-SAFETY-CHECK-PASSED`.
- Has `V62-HEARTBEAT`.
- Has `PHASE2-REGISTERED`.
- If `PHASE2-CONFIRMED` appears, there should be a `PHASE3-CANDIDATE`.
- If `VIRTUAL-POSITION-CLOSE` appears, there should be a `PHASE3-OUTCOME`.
- `virtual_total_closed` and `outcome_total_closed` should stay broadly aligned over time.
- `duplicate_skipped` should stay close to 0.
- `real_execution_enabled=False`.
- `phase3_real_trading_enabled=False`.

## Do Not

- Do not enable `REAL_EXECUTION_ENABLED` in V6.2.
- Do not enable `PHASE3_REAL_TRADING_ENABLED` in V6.2.
- Do not connect V6.2 candidate directly to trader.
- Do not use K-line for Phase2/Phase3 realtime decisions.
- Do not treat V6.2 outcome as proof of profitability without 24-72h run data.
