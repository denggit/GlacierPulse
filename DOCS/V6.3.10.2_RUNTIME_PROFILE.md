# V6.3.10.2 Runtime JSON Profile Auto Loader

## Why

V6.3.10.2 adds a small runtime profile loader so operational settings can live in a JSON file instead of a long shell command. The loader runs at the top of `main.py`, injects supported JSON values into `os.environ`, and then the existing config modules continue to read environment variables exactly as before.

This version only improves runtime configuration. It does not change strategy logic, does not enable A2/A3, and does not enable real trading.

## Default Profile

The default profile path is:

```text
config/runtime_profile.json
```

Run the system directly:

```bash
python main.py
```

If `config/runtime_profile.json` is missing, startup continues with shell environment variables and code defaults.

## Switch Profile

macOS / Linux:

```bash
GLACIER_RUNTIME_PROFILE=config/run_profiles/a1_edge_capture.json python main.py
```

Windows PowerShell:

```powershell
$env:GLACIER_RUNTIME_PROFILE="config/run_profiles/a1_edge_capture.json"
python main.py
```

## Precedence

Default precedence is:

```text
shell env > JSON profile > code defaults
```

This means a shell value such as `REAL_EXECUTION_ENABLED=true python main.py` wins over the JSON file when `allow_override_env` is false. Do not manually override real trading flags to true during A1 research capture.

## allow_override_env

Profiles support:

```json
"allow_override_env": false
```

When false, JSON values do not replace existing shell environment variables. When true, JSON values can overwrite existing shell variables. All checked-in profiles keep this false.

## Value Rules

Supported JSON values are converted into environment strings:

```text
true  -> "true"
false -> "false"
123   -> "123"
1.5   -> "1.5"
"abc" -> "abc"
null  -> skipped
```

Lists and objects are not supported as env values. They are skipped with a `[RUNTIME-PROFILE-WARN]` log line.

## Recommended A1 Edge Capture

The default `config/runtime_profile.json` uses the safe A1 edge capture profile:

```json
{
  "name": "a1_edge_capture",
  "description": "Default safe runtime profile for A1 edge event capture and offline validation",
  "allow_override_env": false,
  "env": {
    "V62_LOG_PROFILE": "RESEARCH_KEY_EVENTS",
    "LOG_TO_CONSOLE": false,
    "LOG_TO_FILE": true,
    "A1_REACTION_RESEARCH_COVERAGE_ENABLED": true,
    "A1_REACTION_EVENT_RECORDER_ENABLED": true,
    "A1_REACTION_EVENT_RECORDER_WRITE_JSONL": true,
    "A1_REACTION_EVENT_RECORDER_JSONL_PATH": "logs/research/a1_reaction_events.jsonl",
    "A1_REACTION_TO_VIRTUAL_POSITION_ENABLED": false,
    "REAL_EXECUTION_ENABLED": false,
    "PHASE3_REAL_TRADING_ENABLED": false,
    "VIRTUAL_SHADOW_MODE": false
  }
}
```

Keep these safety flags false:

```text
REAL_EXECUTION_ENABLED=false
PHASE3_REAL_TRADING_ENABLED=false
A1_REACTION_TO_VIRTUAL_POSITION_ENABLED=false
```

Do not put API keys, secrets, passphrases, or private credentials in runtime profile JSON.

## Confirm It Loaded

Check startup output for:

```text
[RUNTIME-PROFILE]
```

For A1 edge capture, check that the event JSONL is written:

```text
logs/research/a1_reaction_events.jsonl
```

## Analyze After Capture

After collecting events and K-line data, run:

```bash
python tools/analyze_a1_edge.py \
  --events logs/research/a1_reaction_events.jsonl \
  --klines data/ETH-USDT-SWAP_1m.csv \
  --out reports/a1_edge \
  --symbol ETH-USDT-SWAP \
  --random-samples-per-event 10 \
  --seed 42
```
